1 МЕТА
Метою роботи є вивчення та систематизація основних рекомендацій написання чистого коду на мові програмування Go (Golang) відповідно до принципів Clean Code.
Завдання роботи — показати, як принципи чистоти коду (KISS, DRY, YAGNI, Readability) застосовуються в Go, сформувати навички створення структурованого, читабельного та підтримуваного програмного коду з урахуванням стандартів Go та інструментів перевірки стилю.




2 ОПИС ВИКОНАНОЇ РОБОТИ
Вступ
Мова програмування Go, або Golang, була створена у 2007 році в компанії Google трьома інженерами — Робертом Грісемером (Robert Griesemer), Робом Пайком (Rob Pike) та Кеном Томпсоном (Ken Thompson).
Офіційно її представили світу у 2009 році.
Ідея створення Go виникла як реакція на проблеми, з якими стикалися інженери Google при використанні мов C++ та Java — повільна компіляція, надмірна складність та слабка масштабованість.
Go став відповіддю на ці виклики: це компільована, типізована мова, яка поєднує швидкість виконання, простоту синтаксису і зручність паралельних обчислень.
Go активно використовується у таких проєктах, як Docker, Kubernetes, Prometheus, Terraform, і є однією з основних мов для побудови високонавантажених серверів та мікросервісів.
Принципи Clean Code, сформульовані Робертом Мартіном, ідеально узгоджуються з філософією Go — мінімалізм, читабельність, стабільність і простота.


Структура коду
Одним з ключових аспектів чистого коду є правильна структура проєкту.
У Go код організовується в пакети (packages). Кожен пакет має виконувати одну логічну роль, наприклад:
* handlers — для обробки HTTP-запитів,
* services — для бізнес-логіки,
* repositories — для доступу до бази даних.


Це дозволяє розділити відповідальність і підвищує тестованість.


Форматування коду
Мова Go має строгі стандарти форматування, що сприяє єдиному стилю в усіх проєктах.
Інструмент gofmt автоматично вирівнює код і усуває стилістичні відмінності між розробниками.
Правила форматування прості:
* Використовуйте табуляцію, а не пробіли.
* Між функціями вставляйте порожній рядок.
* Довжина рядка не має перевищувати 100 символів.
* Дужки завжди пишуться в одному рядку з оператором if, for, func.
1         func Add(a, b int) int {
2                 return a + b
3        }
4 
5         func Subtract(a, b int) int {
6                 return a - b
7         }
8 
9         func Multiply(a, b int) int {
10                 return a * b
11         }


        Таке форматування дозволяє уникнути плутанини і підвищує читабельність коду навіть для великої команди.
Іменування
Іменування — один з найважливіших елементів чистого коду.
У Go воно має строгі правила:
* CamelCase — для назв типів, структур і функцій.
* lowerCamelCase — для змінних і параметрів функцій.
* Використовуйте зрозумілі імена, які відображають суть об’єкта.
* Не вигадуйте власних скорочень.
1          var userCount int
2          func CalculateSum(numbers []int) int { ... }
3          type UserProfile struct { ... }
        
Хороші імена зменшують потребу в коментарях і спрощують навігацію по коду.


Коментарі та документування
Коментарі в Go використовуються не для пояснення “що робить код”, а для пояснення “чому він це робить”.
GoDoc автоматично формує документацію на основі коментарів, які починаються з імені функції або пакету.
1          // CalculateAverage обчислює середнє значення чисел у масиві.
2          // Повертає 0, якщо масив порожній.
3          func CalculateAverage(numbers []int) float64 { ... }


Коментарі роблять код самодокументованим і підвищують розуміння навіть без додаткової документації.
        Чіткі коментарі зменшують плутанину при читанні коду.
Допомагають під час code review, рефакторингу та підтримки проекту.


Обробка помилок
Go не використовує виключень (try/catch), як інші мови.
Помилка — це звичайне значення, яке функція повертає другим аргументом.
Це змушує розробника явно обробляти помилки і не залишати їх без уваги.
29          data, err := os.ReadFile("config.json")
30          if err != nil {
31                return fmt.Errorf("read config.json: %w", err)
32         }
Такий підхід допомагає швидко знаходити джерело проблеми і зменшує кількість непередбачуваних збоїв.


Інструменти перевірки стилю
Go має потужну екосистему інструментів для контролю якості коду:
* gofmt — форматування;
* golint — перевірка стилю;
* go vet — пошук логічних помилок;
* staticcheck — глибокий аналіз;
* errcheck — перевірка, чи всі помилки були оброблені.
Приклад використання:
gofmt -w .
golint ./...
go vet ./...
staticcheck ./...
У сучасних командах ці перевірки автоматично виконуються через CI/CD перед кожним комітом.


Вплив чистоти коду
Дотримання принципів Clean Code у Go має безпосередній вплив на якість проєкту:
* Зростає читабельність і передбачуваність коду.
* Зменшується кількість багів і технічного боргу.
* Полегшується тестування і рефакторинг.
* Покращується співпраця між розробниками.
Правильно:
8          func CalculateSum(numbers []int) int {
9                  total := 0
10                  for _, n := range numbers {
11                          total += n
12                     }
13                  return total
14           }
Так робити не потрібно:
15        func sum(arr []int) int {total:=0;for i:=0;i<len(arr);i++{total+=arr[i]}return total}
3 ВИСНОВКИ
Під час виконання роботи було досліджено основні принципи написання чистого коду на мові Go відповідно до концепції Clean Code.
Було розглянуто структуру проєкту, форматування, іменування, коментування, обробку помилок та інструменти перевірки стилю.
Дотримання цих рекомендацій дозволяє:
* створювати зрозумілий і підтримуваний код,
* зменшити кількість помилок,
* покращити швидкість розробки,
* забезпечити єдиний стиль у команді.
Таким чином, поєднання принципів Clean Code з особливостями Go забезпечує не лише чистоту коду, але й стабільність, продуктивність і надійність програмного забезпечення.
